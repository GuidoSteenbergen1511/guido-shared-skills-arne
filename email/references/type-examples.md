# Email Type Examples

Complete examples for each of the 8 email types.
Names and projects are generic placeholders — replace with your actual context.

## Type 1: Quick Internal Reply (Dutch, no greeting/closing)

```
Met Sarah en Tim gekeken. Alex inderdaad druk. Gaan het overslaan.
```

## Type 2: Internal Coordination (Dutch)

```
Hoi Mara,

Zouden jullie de klantpresentatie voor volgende week alvast kunnen voorbereiden? Laat even weten of donderdag werkt voor een korte sync.

Groet,
```

## Type 3: Forwarding with Assessment (Dutch)

```
Hoi [collega],

Zie onderstaande mail van de klant over de planningsdatum.

Mijn inschatting:
- Juni 16-18: meest realistisch qua staffing, maar komt dicht op jouw afwezigheid
- September: veiligste optie als we per se willen dat jij erbij bent
- Voorstel: we geven juni door als eerste voorkeur en september als backup

Als jij de data kunt confirmen dan geef ik het door aan de klant.

Groet,
```

## Type 5: External Casual (English)

```
Hi Sarah,

Thanks for sharing the details about your team's current process. The manual report generation sounds time-consuming.

Been thinking about what we discussed regarding automation. Happy to explore this further — a 30-minute call could help us understand the specifics.

Let me know how you'd like to proceed.

Best,
```

## Type 6: External Formal (English, C-suite)

```
Dear Graham,

I hope you had a wonderful break.

I wanted to follow up regarding the programme launch. The first cohort is now live and the momentum has been fantastic.

Looking at the Q2 timeline:
- First cohort week: March 31 - April 2
- Stakeholder interview needed: February (to incorporate your insights before the programme starts)

Your perspective as regional lead remains essential to making sure the content truly reflects the realities of your markets.

Best regards,
```

## Type 7: Decline / Bad News (English)

```
Hi Alex,

Thanks for reaching out about the project. Really appreciate you thinking of us for this.

I've discussed it internally and we've looked at whether we could move some things around to make it work. Unfortunately, the timing just doesn't line up with our current capacity through Q1.

I'd rather be honest with you now than overcommit and underdeliver.

If anything changes on our end, I'll let you know. And if there's anything else we can help with in the meantime, just let me know.

Best,
```

## Type 2: Internal Coordination — Comprehensive Project Status Update (Dutch)

This is the gold standard for multi-topic project updates. Use this structure when bringing a colleague up to speed on a complex project.

```
Hey [collega],

Hieronder een overzicht van waar alles staat op [Projectnaam].

**1. Tooling & Aanpak**

Hier heb ik je al apart over bijgepraat. Kort samengevat: de aanpak is helder — drie gestandaardiseerde workflows en een governance-matrix die we samen gaan opzetten. Het IT-overleg met de klant hebben we vandaag gehad. Details volgen.

**2. [Deliverable X]**

De klant heeft me laten weten dat we vanavond de volledige briefing krijgen. Ik heb gezegd dat ik er morgen eind van de dag op z'n laatst op terugkom.

**3. Communicatie & Programmastatus**

Stuurde zondag een uitgebreide update (jij in CC). Reacties van de klant:

- **[Onderwerp A]:** Klant kiest voor optie B — geen impact op de planning
- **[Datum X]:** Bijna bevestigd. Details volgen van de klant
- **[Onderwerp B]:** Klant pusht erop dat we [Y] actiever bijhouden — moeten we even bespreken

**4. [Deliverables] — Graag checken**

Beide documenten staan klaar op OneDrive:

`Projects/[Klant]/02 Phase 2/`

- **[Document A]** — klaar voor review
- **[Document B]** — klaar voor review

**5. Volgende fase — Op schema**

Check-in staat al in de agenda: **[datum] om [tijd]** met jou en [naam].
```

**Why this works:** Numbered by topic (not chronological). Already-discussed items get 1 line, not a recap. Client responses summarized as bold-labeled bullets. Deliverables with exact file paths. Honest about what's done vs pending. Points to existing calendar check-in. Direct, no fluff, actionable.
